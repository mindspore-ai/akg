这是一个在单次操作中对同一文件进行多处编辑的工具。它构建在 Edit 工具之上，让你能高效执行多次查找替换。需要对同一文件进行多处编辑时，优先使用此工具。

在使用此工具之前：

1. 使用 Read 工具理解文件内容和上下文
2. 验证目录路径是否正确

要进行多处编辑，请提供以下内容：
1. file_path：要修改的文件的绝对路径（必须是绝对路径，不能是相对路径）
2. edits：要执行的编辑操作数组，每个编辑包含：
   - oldString：要替换的文本（必须与文件内容完全匹配，包括所有空白和缩进）
   - newString：用于替换 oldString 的新文本
   - replaceAll：替换 oldString 的所有出现位置。此参数可选，默认 false。

重要：
- 所有编辑按提供的顺序依次应用
- 每次编辑都在前一次编辑的结果上操作
- 所有编辑必须都有效才能成功 —— 若任何一个失败，将不会应用任何修改
- 当你需要对同一文件的不同部分做多处修改时，此工具非常适合

关键要求：
1. 所有编辑都必须遵循单个 Edit 工具的相同要求
2. 编辑是原子的 —— 要么全部成功，要么全部失败
3. 仔细规划编辑，避免顺序应用时产生冲突

警告：
- 如果 edits.oldString 与文件内容不完全匹配（包括空白），工具将失败
- 如果 edits.oldString 与 edits.newString 相同，工具将失败
- 由于编辑按顺序应用，确保较早的编辑不会影响后续编辑要查找的文本

编辑时：
- 确保所有编辑都产生规范、正确的代码
- 不要让代码处于损坏状态
- 始终使用绝对路径（以 / 开头）
- 仅在用户明确要求时才使用表情符号。除非被要求，避免向文件中添加表情符号。
- 如需在整个文件中替换/重命名字符串，请使用 replaceAll。此参数在需要重命名变量时很有用。

如果你要创建新文件，请使用：
- 新文件路径（必要时包含目录）
- 第一次编辑：oldString 为空字符串，newString 为新文件内容
- 后续编辑：对已创建内容进行正常编辑操作
