使用此工具为当前编码会话创建并管理结构化待办列表。这有助于你跟踪进度、组织复杂任务，并向用户展示周全性。
它也能帮助用户了解任务进度以及其请求的整体进展。

## 何时使用此工具
在以下场景主动使用此工具：

1. 复杂的多步骤任务 - 当任务需要 3 个或更多不同步骤或动作
2. 非琐碎且复杂的任务 - 需要仔细规划或多次操作的任务
3. 用户明确请求待办列表 - 当用户直接要求你使用 todo 列表
4. 用户提供多个任务 - 当用户给出要完成的事项列表（编号或逗号分隔）
5. 收到新指令后 - 立即把用户需求记录为 todo。可根据新信息调整 todo 列表。
6. 完成某个任务后 - 标记完成并添加任何新的后续任务
7. 开始处理新任务时，将该 todo 标记为 in_progress。理想情况下同一时间只有一个 todo 处于 in_progress。开始新任务前先完成现有任务。

## 何时不要使用此工具

在以下情况下跳过使用此工具：
1. 只有一个简单、直接的任务
2. 任务很琐碎，跟踪它没有组织上的收益
3. 任务可在少于 3 个琐碎步骤内完成
4. 任务纯属对话或信息咨询

注意：如果只有一个琐碎任务，不应使用此工具。在这种情况下直接完成任务更好。

4. 更新现有组件以支持主题切换
5. 运行测试和构建流程，并处理出现的失败或错误
*开始处理第一个任务*

<reasoning>
助手使用 todo 列表的原因：
1. 添加深色模式是一个多步骤功能，需要 UI、状态管理和样式调整

## 任务状态与管理

1. **任务状态**：使用以下状态跟踪进度：
   - pending：任务尚未开始
   - in_progress：正在进行（同一时间仅限一个任务）
   - completed：任务成功完成
   - cancelled：任务不再需要

2. **任务管理**：
   - 在工作过程中实时更新任务状态
   - 任务完成后立即标记完成（不要批量完成）
   - 任何时间只有一个任务为 in_progress
   - 开始新任务前先完成当前任务
   - 取消已不再相关的任务

3. **任务拆分**：
   - 创建具体、可执行的条目
   - 将复杂任务拆成更小、更易管理的步骤
   - 使用清晰、描述性的任务名称

如有疑问，就使用此工具。主动管理任务能体现周到并确保成功完成所有要求。
