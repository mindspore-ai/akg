# 任务特征
**操作类型**：broadcast类型，broadcast第二根轴；2D Tensor输入，2D Tensor输出
**数据尺寸**：(2048, 131072)、(2048, 1)，broadcast轴中等，非broadcast轴shape大
**数据类型**：float16
**任务特点**：操作类型为broadcast，broadcast轴中等，非broadcast轴shape大，可以按照轴的顺序，外层并行，内层向量化，若UB存不下，可考虑多次切分。

## 优化
**优化内容**：对于一般的2Dshape，通用的调度方法是把第一维（M维）映射到多核上实现并行，除非第一维很小。
1. **核间并行（NUM_BLOCKS）**：沿M维度切分，分配到不同计算核（具体核数可以通过autotune进行尝试调优），充分利用硬件并行能力。
2. **核内行切分（SUB_M）**：在每个核内部进一步切分M维度，控制每次处理的行数，平衡UB利用率和寄存器压力。
3. **列向量化（BLOCK_N）**：沿N维度分块加载，每次一次性搬运BLOCK_N个连续元素（如16384个），实现内存访问连续性和向量化操作，同时避免一次性加载整行导致UB溢出。
